# 🚧 Features & Roadmap

The goal of deadrop is to provide platform and device agnostic secret sharing in a streamlined and easy-to-use fashion. This page tracks what's shipped, what's in experimental rollout, and what's coming next.

Status key:

- ✅ **Shipped**: stable and supported
- 🧪 **Experimental**: live and usable, expect breaking changes between releases
- 🛠️ **In progress**: actively being built
- 📋 **Planned**: in the backlog, accepting input

For per-surface usage, see the [CLI](/docs/features/cli) and [VS Code](/docs/features/vscode) docs.

### Sharing & handoff

- **Raw text drops**: paste or pipe a string, drop it to a single recipient
- **JSON payloads**: structured key/value drops with the same protocol
- **File drops**: drop the contents of a file up to the per-drop size limit
- **End-to-end encryption**: ECDH key exchange, AES-256-GCM payloads, SHA-256 integrity verification
- **WebRTC peer-to-peer transport**: secrets never transit our servers, only opaque session IDs

### Authentication & abuse prevention

- **Account-optional usage**: drop and grab work without an account
- **Clerk-backed sign-in**: required for premium features
- **hCaptcha**: lightweight abuse protection on anonymous drops

### Surfaces

- **Web app**: `deadrop.io`, installable as a PWA
- **CLI**: drop, grab, vault, secret, inject commands. Available on npm and via the install script

### CI/CD

- **`deadrop inject`**: run a command with a vault's secrets injected into its environment, no file on disk. See the [CLI docs](/docs/features/cli) for usage.

### CLI authentication

- **`deadrop login` / `logout` / `whoami`**: Clerk-backed CLI auth flow. Tokens are cached in the OS keychain (macOS Keychain, Linux Secret Service, Windows Credential Vault), never written to disk in plaintext. [#25](https://github.com/dallen4/deadrop/issues/25)

### Editor integration

- **VS Code extension** (Supporter+): live on the [VS Code Marketplace](https://marketplace.visualstudio.com/search?term=deadrop&target=VSCode). Drop, grab, and browse vaults without leaving your editor. Actively iterating, so expect breaking changes between releases. <br/>[Track on GitHub →](https://github.com/dallen4/deadrop/issues)

### Vaults

- **Local vaults** (web): browser-based encrypted vault using OPFS. CLI version is stable. [#58](https://github.com/dallen4/deadrop/issues/58)
- **Cloud-synced vaults** (Supporter+): encrypted Turso replica for cross-device access. [#58](https://github.com/dallen4/deadrop/issues/58)

### Multidrop

- **Multi-recipient drops** (Supporter+): send the same secret to multiple grabbers in one session. Free plan is capped at 1 grabber. [#6](https://github.com/dallen4/deadrop/issues/6)

### CI/CD

- **CI service tokens**: `deadrop inject` mints short-lived, read-only vault tokens for non-interactive build pipelines (via `DEADROP_API_KEY` and config-free env vars), replacing the long-lived token in a CI config. Currently gated to early-access accounts.

### Sharing

- **Configurable drop options**: expiry, file size limits, and other per-drop settings. [#4](https://github.com/dallen4/deadrop/issues/4)
- **Drop passcode protection**: require a passphrase or 2FA confirmation to grab, in addition to the drop ID. [#5](https://github.com/dallen4/deadrop/issues/5)
- **Dynamic drop links**: shareable short links that auto-generate a drop session. [#61](https://github.com/dallen4/deadrop/issues/61)
- **Dynamic QR codes**: rotating QR for short-lived in-person handoffs.

### Collaboration

- **Read-only sharing for humans** (Pro+): grant collaborators view-only access to vault entries.
- **Write delegation** (Pro+): allow named collaborators to update specific vault entries.

### CI/CD

- **GitHub Action**: composite action wrapping `deadrop inject`, with masked log output. GitLab CI template and CircleCI orb to follow.

### Distribution

- **Homebrew formula**: `brew install deadrop` for macOS users. [#21](https://github.com/dallen4/deadrop/issues/21)

### Integrations

- **Slack app**: drop and grab from Slack without leaving your workspace. [#11](https://github.com/dallen4/deadrop/issues/11)

### Enterprise

- **SSO (SAML / OIDC)** (Org): enterprise identity provider integration.
- **Role-based env access** (Org): per-environment RBAC.
- **Audit log + export** (Org): full activity log with CSV/JSON export.
